# Shakass Lab - Instructions Claude Code

## Objectif du projet
Shakass Lab est un module HTML/CSS/JS premium de personnalisation textile.
Le but n’est pas de faire un configurateur banal, mais une interface haut de gamme, fluide, moderne, type Photoshop simplifié pour T-shirt.

## Périmètre
Ce projet concerne uniquement Shakass Lab / Shakass Com.
Ne pas mélanger avec TSL, WinShirt Shopify, app Shopify Custom ou TextileLab backend.

## Contraintes design
- Interface premium, moderne, claire, effet waouh.
- Desktop et mobile doivent être vraiment travaillés.
- L’UX doit être simple malgré la richesse fonctionnelle.
- Le configurateur doit donner une impression de produit sérieux, pas de prototype bricolé.
- Priorité à la lisibilité, fluidité, responsive, animations propres.

## Fonctionnalités attendues
- Zone centrale produit/mockup.
- Sidebar ou menus outils : texte, image, SVG, QR code, calques, formats, couleurs.
- Drag & drop, resize, rotation si déjà présent ou prévu.
- Gestion mobile propre, pas juste une version écrasée.
- Aucun chevauchement avec footer ou sections voisines.
- Code propre, maintenable, sans CSS inutilement dupliqué.

## Méthode obligatoire
Avant toute modification :
1. Auditer les fichiers existants.
2. Identifier les bugs bloquants.
3. Écrire un plan court.
4. Corriger par petits lots.
5. Tester en local.
6. Vérifier desktop + mobile.
7. Faire un rapport clair des changements.

Ne jamais dire que c’est terminé sans preuve :
- serveur local lancé,
- page visible,
- aucune erreur console critique,
- responsive testé,
- checklist validée.

## Commandes
Si package.json existe, utiliser les scripts disponibles.
Sinon lancer un serveur local simple avec Python ou npx serve selon ce qui existe sur la machine.
